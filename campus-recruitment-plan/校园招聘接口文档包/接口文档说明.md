# 校园智能招聘与面试预约平台：接口文档 / OpenAPI 草案说明

版本：v1.0  
OpenAPI 版本：3.1.0  
基础路径：`/api`  
认证方式：`Authorization: Bearer <token>`  
默认响应格式：统一 JSON 响应

---

## 1. 文档说明

本文档是“校园智能招聘与面试预约平台”的接口设计文档，配套文件为：

```text
OpenAPI接口草案.yaml
```

该 YAML 文件可导入：

- Apifox
- Swagger UI
- Swagger Editor
- Postman
- Knife4j / springdoc-openapi 后续对齐

本文档用于给开发、测试、前端联调使用；`OpenAPI接口草案.yaml` 用于工具导入和接口 mock。

## 2. 全局规范

### 2.1 请求规范

| 项目 | 规范 |
|---|---|
| 协议 | HTTP / HTTPS |
| 数据格式 | JSON |
| 文件上传 | `multipart/form-data` |
| 字符编码 | UTF-8 |
| 时间格式 | ISO 8601，例如 `2026-05-20T10:00:00` |
| 认证头 | `Authorization: Bearer <token>` |

### 2.2 通用响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|---|---|---|
| `code` | integer | 业务状态码，0 表示成功 |
| `message` | string | 响应消息 |
| `data` | object | 响应数据 |

### 2.3 分页响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "records": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

### 2.4 常用 HTTP 状态码

| 状态码 | 含义 | 使用场景 |
|---:|---|---|
| 200 | OK | 查询、修改成功 |
| 201 | Created | 创建成功 |
| 400 | Bad Request | 参数错误 |
| 401 | Unauthorized | 未登录或 Token 失效 |
| 403 | Forbidden | 无权限 |
| 404 | Not Found | 资源不存在 |
| 409 | Conflict | 重复投递、重复预约等业务冲突 |
| 500 | Internal Server Error | 系统异常 |

## 4. 关键接口详细说明

## 4.1 注册

`POST /api/auth/register`

### 请求体

```json
{
  "username": "zhangsan",
  "password": "123456",
  "confirmPassword": "123456",
  "userType": "STUDENT",
  "phone": "13800000000",
  "email": "test@example.com"
}
```

### 响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "userId": 10001,
    "username": "zhangsan",
    "userType": "STUDENT"
  }
}
```

### 业务规则

| 编号 | 规则 |
|---|---|
| AUTH-001 | 用户名唯一 |
| AUTH-002 | 密码和确认密码必须一致 |
| AUTH-003 | `userType` 只能是 `STUDENT` 或 `COMPANY` |
| AUTH-004 | 密码必须加密存储 |

---

## 4.5 投递岗位

`POST /api/applications`

### 请求体

```json
{
  "jobId": 1,
  "resumeId": 1
}
```

### 业务规则

| 编号 | 规则 |
|---|---|
| APP-001 | 学生必须登录 |
| APP-002 | 岗位必须是 `PUBLISHED` |
| APP-003 | 如果不传 `resumeId`，使用默认简历 |
| APP-004 | 同一学生不能重复投递同一岗位 |
| APP-005 | 投递成功后发送 MQ 通知 |

---

## 4.6 预约面试

`POST /api/interview/bookings`

### 请求体

```json
{
  "slotId": 1,
  "applicationId": 1
}
```

### 业务规则

| 编号 | 规则 |
|---|---|
| INT-001 | 学生必须获得面试邀约 |
| INT-002 | 场次必须是 `OPEN` |
| INT-003 | 剩余名额必须大于 0 |
| INT-004 | 使用 Redis Lua 扣减库存 |
| INT-005 | 数据库唯一索引防重复预约 |
| INT-006 | 预约成功后发送 MQ 通知 |
| INT-007 | DB 写入失败时需要库存补偿 |

## 6. OpenAPI 文件说明

配套 `OpenAPI接口草案.yaml` 已包含：

- `info`
- `servers`
- `tags`
- `securitySchemes`
- `paths`
- `components.schemas`
- 通用响应结构
- 主要请求体和响应体
- 主要业务枚举
- Bearer Token 认证配置

建议后续开发时：

1. 后端用 springdoc-openapi 自动生成真实接口文档。
2. 将真实接口与本草案进行 diff。
3. 前端/测试使用 Apifox 导入 `OpenAPI接口草案.yaml`。
4. 随着接口变更同步维护该文件。
